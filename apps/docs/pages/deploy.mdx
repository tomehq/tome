# Deployment

Tome builds to static HTML, CSS, and JS. You can deploy the output anywhere -- Tome Cloud, Vercel, Netlify, Cloudflare Pages, or any web server.

## Build for Production

```bash
tome build
```

This produces a `out/` directory (configurable with `-o`) containing your entire docs site as static files. No server runtime required.

## Tome Cloud

Tome Cloud is the fastest way to deploy. One command, incremental uploads, and automatic SSL.

<Steps>
  <div>
    **Authenticate**

    Run `tome login` to connect your Tome Cloud account. This opens your browser for authentication and saves a token to `~/.tome/config`.

    ```bash
    tome login
    ```
  </div>
  <div>
    **Build your site**

    ```bash
    tome build
    ```
  </div>
  <div>
    **Deploy**

    ```bash
    tome deploy
    ```

    Tome computes SHA-256 hashes for every file and sends a manifest to the server. Only files that changed since the last deploy are uploaded. Your site is live within seconds.
  </div>
</Steps>

After deploying, your site is available at `https://<slug>.tome.dev`. You can also connect a custom domain -- see the Custom Domains guide.

### Incremental Uploads

Tome tracks file changes by content hash. On each deploy:

1. The CLI hashes every file in the build output
2. The manifest is sent to Tome Cloud
3. The server compares hashes against the previous deployment
4. Only new or changed files are uploaded

This means subsequent deploys are fast -- typically under a few seconds for content-only changes.

## Self-Hosting

The `out/` directory is plain static files. Deploy it to any platform that serves HTML.

<Tabs items={["Vercel", "Netlify", "Cloudflare Pages", "Static Server"]}>
  <div>
    ```bash
    # Install Vercel CLI
    npm i -g vercel

    # Deploy
    vercel deploy ./out
    ```

    Or connect your Git repository and set the build command to `tome build` with the output directory set to `out`.
  </div>
  <div>
    ```bash
    # Install Netlify CLI
    npm i -g netlify-cli

    # Deploy
    netlify deploy --dir=out --prod
    ```

    Or connect your Git repository. Set the build command to `tome build` and the publish directory to `out`.
  </div>
  <div>
    ```bash
    # Install Wrangler
    npm i -g wrangler

    # Deploy
    wrangler pages deploy ./out
    ```

    Or connect your Git repository in the Cloudflare dashboard. Set the build command to `tome build` and the output directory to `out`.
  </div>
  <div>
    The `out/` directory can be served by any HTTP server:

    ```bash
    # Python
    cd out && python3 -m http.server 8000

    # Node (serve)
    npx serve out

    # Nginx, Apache, Caddy -- point the document root to out/
    ```

    You can also upload the files to S3, GCS, or any object storage with static site hosting enabled.
  </div>
</Tabs>

## CI/CD

Automate deployments by running `tome build && tome deploy` in your CI pipeline. Use the `--token` flag with `tome login` to authenticate non-interactively:

```bash
# In CI environment
tome login --token $TOME_API_TOKEN
tome build
tome deploy
```

Store your API token as a CI secret (e.g., `TOME_API_TOKEN` in GitHub Actions, GitLab CI, or your CI platform's secrets manager).
