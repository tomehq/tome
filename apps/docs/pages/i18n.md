# Internationalization

Tome supports multi-locale documentation through the `i18n` configuration. Each locale gets its own set of pages with automatic fallback to the default locale for untranslated content.

## Configuration

Enable i18n in `tome.config.js`:

```javascript
export default {
  i18n: {
    defaultLocale: "en",
    locales: ["en", "es", "ja"],
    localeNames: {
      en: "English",
      es: "Espanol",
      ja: "Japanese",
    },
    fallback: true,
  },
};
```

## File Structure

Organize translated pages in locale subdirectories under `pages/`:

```text
pages/
├── en/
│   ├── index.md
│   ├── quickstart.md
│   └── configuration.md
├── es/
│   ├── index.md
│   └── quickstart.md        # Only translated pages needed
├── ja/
│   └── index.md
└── index.md                  # Fallback for default locale
```

Pages in the default locale directory (`en/`) can also be placed at the root of `pages/` -- both locations work.

## Locale Fallback

When `fallback: true` (the default), untranslated pages fall back to the default locale. If a user visits `/es/configuration` but `pages/es/configuration.md` does not exist, Tome serves `pages/en/configuration.md` instead.

Set `fallback: false` to return a 404 for missing translations.

## URL Structure

Locale-prefixed URLs are used for all non-default locales:

| Locale | URL |
|--------|-----|
| `en` (default) | `/quickstart` |
| `es` | `/es/quickstart` |
| `ja` | `/ja/quickstart` |

The default locale does not have a prefix.

## Navigation

Navigation is shared across locales. Tome uses the same `navigation` config for all locales, resolving page IDs to the appropriate locale directory. Translated pages use localized frontmatter for sidebar titles.

## Config Reference

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `i18n.defaultLocale` | `string` | `"en"` | Default locale code |
| `i18n.locales` | `string[]` | `["en"]` | All supported locale codes |
| `i18n.localeNames` | `Record<string, string>` | -- | Display names for each locale |
| `i18n.fallback` | `boolean` | `true` | Fall back to default locale for missing translations |
