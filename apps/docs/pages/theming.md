# Theming

Tome's theming system is controlled through the `theme` field in `tome.config.js`. You can use a built-in preset as-is, override specific values, or fully customize colors and fonts.

## Theme Presets

Tome ships with two presets that define the complete color palette and font stack:

### Amber (default)

The default preset. Warm amber accent on a neutral background, with serif headings and a clean sans-serif body.

```javascript
export default {
  theme: {
    preset: "amber",
  },
};
```

**Fonts:**
- Headings: Instrument Serif
- Body: DM Sans
- Code: JetBrains Mono

### Editorial

A bold, editorial-inspired theme with a coral accent and deep navy dark mode. Designed for content-heavy sites.

```javascript
export default {
  theme: {
    preset: "editorial",
  },
};
```

**Fonts:**
- Headings: Cormorant Garamond
- Body: Bricolage Grotesque
- Code: Fira Code

## Custom Accent Color

Override the accent color on any preset by providing a hex color code:

```javascript
export default {
  theme: {
    preset: "amber",
    accent: "#6366f1", // Indigo accent
  },
};
```

The accent color is used for:
- Active sidebar links
- Link text
- Inline code highlighting
- Callout borders (for default callouts)
- Step number badges
- Tab indicators

Tome automatically generates dimmed and hover variants from the accent color. In dark mode, the dimmed variant uses 12% opacity; in light mode, 8% opacity.

The accent must be a valid six-character hex code (e.g., `#e8a845`). Shorthand hex codes are not supported.

## Color Mode

The `mode` field controls whether the site renders in light mode, dark mode, or follows the user's system preference.

```javascript
export default {
  theme: {
    mode: "auto", // "light" | "dark" | "auto"
  },
};
```

| Mode | Behavior |
|------|----------|
| `"auto"` | Matches the operating system's color scheme. A toggle button appears in the sidebar footer so users can switch manually. |
| `"light"` | Always light mode. No toggle button. |
| `"dark"` | Always dark mode. No toggle button. |

When `mode` is set to `"auto"`, Tome listens for changes to the `prefers-color-scheme` media query and updates in real time if the user changes their system settings.

## Custom Fonts

Override any of the three font slots -- heading, body, or code:

```javascript
export default {
  theme: {
    preset: "amber",
    fonts: {
      heading: "Inter",
      body: "Inter",
      code: "Fira Code",
    },
  },
};
```

Each font slot maps to a CSS custom property:

| Slot | CSS Variable | Used For |
|------|-------------|----------|
| `heading` | `--font-heading` | Page titles (`h1`), sidebar name |
| `body` | `--font-body` | Body text, sidebar links, UI elements |
| `code` | `--font-code` | Code blocks, inline code, keyboard shortcuts |

If you use a Google Font, make sure it is loaded in your `index.html`. The default `index.html` generated by `tome init` includes the fonts used by the `amber` preset. If you switch to a different font, update the `<link>` tags accordingly.

### Example: Loading a Custom Google Font

```html
<link
  href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&display=swap"
  rel="stylesheet"
/>
```

If you only override one slot, the others inherit from the preset:

```javascript
export default {
  theme: {
    preset: "editorial",
    fonts: {
      code: "JetBrains Mono", // Override just the code font
    },
  },
};
```

## Color Tokens

Each preset defines a set of color tokens for both light and dark mode. These are applied as CSS custom properties on the root element:

| Token | Variable | Description |
|-------|----------|-------------|
| Background | `--bg` | Page background |
| Surface | `--sf` | Card and panel backgrounds |
| Surface Hover | `--sfH` | Hovered surface elements |
| Border | `--bd` | Borders and dividers |
| Text | `--tx` | Primary text |
| Text Secondary | `--tx2` | Body text and descriptions |
| Text Muted | `--txM` | Subtle labels and hints |
| Accent | `--ac` | Primary accent color |
| Accent Dim | `--acD` | Low-opacity accent fill |
| Accent Tint | `--acT` | Accent hover state |
| Code Background | `--cdBg` | Code block background |
| Code Text | `--cdTx` | Code block text |
| Sidebar Background | `--sbBg` | Sidebar background |
| Header Background | `--hdBg` | Header background (with blur) |

These tokens are set automatically based on your preset and accent color. In most cases you do not need to interact with them directly, but they are available if you write custom CSS.

## Putting It All Together

Here is a complete theme configuration that uses the editorial preset with a custom teal accent, auto color mode, and a custom code font:

```javascript
import { defineConfig } from "@tome/core";

export default defineConfig({
  name: "My Docs",
  theme: {
    preset: "editorial",
    accent: "#0d9488",
    mode: "auto",
    fonts: {
      code: "JetBrains Mono",
    },
  },
});
```
