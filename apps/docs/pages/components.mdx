# Built-in Components

Tome ships with a set of MDX components that are automatically available in any `.mdx` file. No imports needed -- just use them directly in your markup.

These components only work in `.mdx` files, not plain `.md` files.

## Callout

Callouts draw attention to important information. They support four types: `info`, `warning`, `tip`, and `danger`.

### Info

<Callout type="info" title="Note">
  This is an informational callout. Use it for supplementary details or context that readers might find helpful.
</Callout>

```jsx
<Callout type="info" title="Note">
  This is an informational callout.
</Callout>
```

### Warning

<Callout type="warning" title="Warning">
  This is a warning callout. Use it when the reader should proceed with caution or be aware of potential issues.
</Callout>

```jsx
<Callout type="warning" title="Warning">
  This is a warning callout.
</Callout>
```

### Tip

<Callout type="tip" title="Pro tip">
  This is a tip callout. Use it for best practices, shortcuts, or helpful suggestions.
</Callout>

```jsx
<Callout type="tip" title="Pro tip">
  This is a tip callout.
</Callout>
```

### Danger

<Callout type="danger" title="Danger">
  This is a danger callout. Use it for critical warnings about data loss, security risks, or breaking changes.
</Callout>

```jsx
<Callout type="danger" title="Danger">
  This is a danger callout.
</Callout>
```

### Without a Title

The `title` prop is optional. Without it, the callout renders just the body text:

<Callout type="info">
  A callout without a title. The content stands on its own.
</Callout>

```jsx
<Callout type="info">
  A callout without a title.
</Callout>
```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `type` | `"info" \| "warning" \| "tip" \| "danger"` | `"info"` | Visual style and icon. |
| `title` | `string` | -- | Optional heading displayed above the body. |
| `children` | `ReactNode` | -- | The callout body content. |

## Tabs

Tabs let readers switch between related content panels -- useful for showing examples in multiple languages or comparing approaches.

<Tabs items={["npm", "yarn", "pnpm"]}>
  <div>
    ```bash
    npm install @tome/cli
    ```
  </div>
  <div>
    ```bash
    yarn add @tome/cli
    ```
  </div>
  <div>
    ```bash
    pnpm add @tome/cli
    ```
  </div>
</Tabs>

```jsx
<Tabs items={["npm", "yarn", "pnpm"]}>
  <div>npm install @tome/cli</div>
  <div>yarn add @tome/cli</div>
  <div>pnpm add @tome/cli</div>
</Tabs>
```

### Props

| Prop | Type | Description |
|------|------|-------------|
| `items` | `string[]` | Labels for each tab. |
| `children` | `ReactNode[]` | Content panels. The number of children must match the number of items. |

### Card Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `string` | -- | Card heading (required). |
| `icon` | `string` | -- | An emoji or icon string displayed above the title. |
| `href` | `string` | -- | Makes the card a clickable link. |
| `children` | `ReactNode` | -- | Description text below the title. |

| Prop | Type | Default | Description |
|------|------|---------|-------------|

## Steps

Steps display a numbered sequence of instructions with a vertical timeline.

<Steps>
  <div>
    **Create a project**

    Run `npx @tome/cli init my-docs` to scaffold a new documentation site.
  </div>
  <div>
    **Install dependencies**

    Navigate into the project directory and run `npm install`.
  </div>
  <div>
    **Start writing**

    Open `pages/index.md` and write your content. The dev server reloads on save.
  </div>
</Steps>

```jsx
<Steps>
  <div>
    **Create a project**

    Run `npx @tome/cli init my-docs` to scaffold a new site.
  </div>
  <div>
    **Install dependencies**

    Navigate into the project and run `npm install`.
  </div>
  <div>
    **Start writing**

    Open `pages/index.md` and edit your content.
  </div>
</Steps>
```

### Props

| Prop | Type | Description |
|------|------|-------------|
| `children` | `ReactNode` | Each direct child becomes a numbered step. |

## Accordion

Accordions collapse content behind a clickable header. Useful for FAQs, optional details, or long reference content.

<Accordion title="How do I add a new page?">
  Create a `.md` or `.mdx` file in the `pages/` directory. The filename determines the URL path. Add the page ID to the `navigation` array in `tome.config.js` to make it appear in the sidebar.
</Accordion>

<Accordion title="Can I use custom React components?">
  Yes. Create a `.mdx` file and import your components at the top of the file, or pass them through the MDX component map. The built-in components (Callout, Tabs, etc.) are injected automatically.
</Accordion>

<Accordion title="Does Tome support dark mode?">
  Yes. Set `theme.mode` to `"auto"` (follows system preference), `"dark"`, or `"light"` in your `tome.config.js`. When mode is `"auto"`, a toggle button appears in the sidebar footer.
</Accordion>

```jsx
<Accordion title="How do I add a new page?">
  Create a `.md` or `.mdx` file in the `pages/` directory.
  Add the page ID to the navigation array in tome.config.js.
</Accordion>
```

### Props

| Prop | Type | Description |
|------|------|-------------|
| `title` | `string` | The clickable header text. |
| `children` | `ReactNode` | Content revealed when the accordion is expanded. |
