# Search

Tome includes search out of the box. No configuration needed for local search -- it works immediately. For production sites with large doc sets, you can switch to Algolia DocSearch.

## Local Search

Local search is the default. It uses Pagefind to build a search index at build time and runs entirely in the browser. No external services, no API keys.

```javascript
export default {
  search: {
    provider: "local",
  },
};
```

You can omit the `search` field entirely -- local search is the default.

### How it works

1. During `tome build`, Pagefind indexes all page content, headings, and tags
2. The search index is bundled as static files in the build output
3. At runtime, the search UI queries the index client-side
4. Results include page title, matched excerpt, and heading anchors

Local search is fast and private -- no data leaves the user's browser.

## Algolia DocSearch

For larger documentation sites, Algolia DocSearch provides faster full-text search with typo tolerance, faceted filtering, and analytics.

<Steps>
  <div>
    **Generate crawler config**

    ```bash
    tome algolia:init -u https://docs.example.com
    ```

    This creates a `docsearch-config.json` file configured for your site's page structure and heading hierarchy.
  </div>
  <div>
    **Apply to DocSearch**

    Submit your site at [docsearch.algolia.com](https://docsearch.algolia.com/apply/) or set up your own Algolia account. You will receive an Application ID, Search API Key, and Index Name.
  </div>
  <div>
    **Configure Tome**

    Add the Algolia credentials to your `tome.config.js`:

    ```javascript
    export default {
      search: {
        provider: "algolia",
        appId: "YOUR_APP_ID",
        apiKey: "YOUR_SEARCH_API_KEY",
        indexName: "YOUR_INDEX_NAME",
      },
    };
    ```
  </div>
</Steps>

<Callout type="tip" title="Search-only key">
  Use a search-only API key in your config. This key is embedded in the client-side JavaScript and is safe to expose. Never use your admin API key here.
</Callout>

### Config Reference

| Field | Type | Description |
|-------|------|-------------|
| `search.provider` | `"local" \| "algolia"` | Search backend (default: `"local"`) |
| `search.appId` | `string` | Algolia Application ID |
| `search.apiKey` | `string` | Algolia search-only API key |
| `search.indexName` | `string` | Algolia index name |
